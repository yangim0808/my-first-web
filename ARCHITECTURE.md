# 블로그 프로젝트 아키텍처 (ARCHITECTURE.md)

## 1. 프로젝트 목표
이 프로젝트는 개인 블로그를 위한 정적 사이트를 구축하는 것을 목표로 합니다.
- **성능:** Next.js의 App Router 기능을 활용한 빠른 페이지 로딩.
- **유지보수:** 컴포넌트 분리를 통한 코드의 재사용성과 유지보수성 향상.
- **디자인:** Tailwind CSS와 shadcn/ui 기반의 현대적이고 일관된 UI 제공.
- **확장성:** 향후 데이터베이스 연동 및 기능 추가(인증, 검색 등)에 유연하게 대응할 수 있는 구조.

## 2. 페이지 맵 (URL 구조)
현재 웹 애플리케이션의 URL 구조 및 각 페이지의 역할입니다.

- `/` : 메인 페이지 (최근 게시글 목록 표시 및 짧은 소개)
- `/about` : 소개 페이지 (블로그 및 운영자 소개)
- `/contact` : 연락처 페이지 (연락 가능한 수단 안내)
- `/posts` : 전체 글 목록 페이지
- `/posts/[id]` : 개별 게시글 상세 페이지 (글 내용을 보여주는 상세뷰)
- `/admin` : 관리자 페이지 (글 작성 시작, 관리)
  - `/admin/new` : 새 글 작성 페이지
  - `/admin/edit/[id]` : 글 수정 페이지

1. **메인 페이지(`/`) 접속:** 사용자가 블로그에 접속하여 최근 글 목록을 확인.
2. **글 목록 조회:** `/posts` 페이지로 이동하여 전체 글 목록 탐색 가능, 검색(도입 예정) 활용.

4. **글 읽기:** 글 내용을 읽고 다른 글로 네비게이팅 하거나 목록으로 돌아감.

## 4. 컴포넌트 구조
UI 구성 요소는 기본적으로 `components/ui/` 경로에 설치된 shadcn/ui 컴포넌트를 베이스로 사용하며, 이를 조합하여 `components/` 폴더 내에 커스텀 컴포넌트를 구성합니다.

- **Button (`components/ui/button.tsx`)**
  - **활용 위치:** 폼 제출, 주요 네비게이션 이동, 글 작성/발행 버튼 및 모달 내 확인/취소 동작에 사용.
- **Card (`components/ui/card.tsx`)**
  - **활용 위치:** 메인 페이지 및 목록 페이지(`/posts`)의 개별 게시글 요약 컨테이너, 작성자 정보 패널 등.
- **Input (`components/ui/input.tsx`)**
  - **활용 위치:** 새 글 작성(`/admin/new`) 시 제목 입력, 검색바 텍스트 입력장 및 폼 데이터 수집 영역.
- **Dialog (`components/ui/dialog.tsx`)**
  - **활용 위치:** 게시글 삭제 시 최종 확인 모달창, 사용자에게 중요한 경고나 선택을 요구하는 팝업 창.

## 5. 데이터 모델
데이터베이스 연동 시 사용할 관계형 데이터 모델의 기본 설계 구조입니다.

### 5.1. Users (사용자 테이블)
블로그 관리자 및 향후 가입될 독자 정보를 관리합니다.
- `id` (UUID / Integer) : Primary Key
- `email` (String) : 고유 이메일 주소 (Unique)
- `name` (String) : 표시 이름
- `created_at` (Timestamp) : 가입 일시

### 5.2. Posts (게시글 테이블)
사용자가 작성한 블로그 글의 데이터를 관리합니다.
- `id` (UUID / Integer) : Primary Key
- `title` (String) : 게시글 제목
- `content` (Text) : 게시글 본문 내용
- `author_id` (UUID / Integer) : 작성자 식별자 (Foreign Key 참조 -> `Users.id`)
- `created_at` (Timestamp) : 최초 작성 일시
- `updated_at` (Timestamp) : 최종 수정 일시

### 5.3. 테이블 관계 (Relationships)
- **1 : N 관계 (Users : Posts)**
  - 하나의 `User`는 여러 개의 `Post`를 작성할 수 있습니다.
  - 하나의 `Post`는 반드시 하나의 `User`(`author_id`)에 종속됩니다.
